## Tips

- **Use conventional commit type**: After gitmoji, include type (feat/fix/refactor/etc.)
- **Be specific with scope**: `feat(settings):` not just `feat:`
- **One gitmoji per commit**: Don't combine multiple (e.g., âœ¨ğŸ›)
- **Match commit body**: If you use âš¡, explain the performance improvement
- **Consider changelog**: Will users care? If yes, use user-facing gitmoji
